# What Is a Chunk?

**TL;DR:** A chunk is a small slice of a document — a few hundred words — kept
together with the name of the file it came from.

## Why not use the whole document?

Imagine one file covering black holes, the solar system, and rockets all at
once. If you embed the whole thing into one vector, that vector has to represent
all three topics at the same time. It becomes a blurry average. When someone
asks about event horizons, the retrieval pulls back the whole doc — most of it
irrelevant.

Chunks fix this: each chunk is about **one focused idea**. Its embedding is a
sharp, specific signal, not a blurry average.

## What a chunk looks like in code

```python
Chunk {
    id:       "node_a1b2c3",
    text:     "The event horizon is the boundary of a black hole, not a physical surface...",
    source:   "black_holes.md",
    metadata: { "file_path": "data/sample_docs/black_holes.md" }
}
```

- `text` — the actual slice of document text (~380 words / 512 tokens)
- `source` — which file it came from. This is what powers **citations** later.
  When the model answers *"nothing can escape the event horizon [1]"*, the `[1]`
  maps back to this `source` field.
- `id` — unique identifier generated by LlamaIndex

## How chunks are created

```python
# docsmind/ingestion/chunker.py
splitter = SentenceSplitter(chunk_size=512, chunk_overlap=64)
nodes = splitter.get_nodes_from_documents(documents)
```

- `chunk_size=512` → up to 512 **tokens** per chunk (~380 words)
- `chunk_overlap=64` → see [why-overlap.md](why-overlap.md)
- **Sentence-aware:** it tries to break at sentence boundaries so text stays readable

## After chunking

The sample corpus has 5 space docs (black holes, the solar system, rockets, the
ISS, stellar life cycles), each short enough to produce one chunk — so you end up
with 5 chunks. With longer documents you'd get many chunks per file. Each chunk
will be:
1. Embedded → turned into 384 numbers
2. Stored in FAISS → made searchable
3. Retrieved → matched to a question
4. Sent to the LLM → used as grounding context

The `source` field travels with the chunk through all these steps so citations
always trace back to the right file.
